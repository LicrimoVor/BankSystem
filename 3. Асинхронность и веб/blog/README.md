## Стэк

В данном проекте мы использовали следующий стек технологий:
- SeaORM - СУБД и миграции
- Axum - HTTP API framework
- Tonic - gRPC framework
- Memory State и PostgreSQL - хранение состояния

### Авторизация 
Используется синтез двух токенов: JWT и refresh. При авторизации в куки клиента сохраняется refresh-токен, а в ответе дается JWT-токен.

Клиент должен помнить хранить JWT-токен (в памяти). Если токен отсутствует (например, после перезагрузки страницы или долгой сессии), клиент отправляет запрос с refresh-токеном, чтобы получить новый JWT-токен.
